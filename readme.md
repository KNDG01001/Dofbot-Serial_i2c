DOFBOT PC Control via Arduino Bridge (I2C)이 프로젝트는 Jetson Nano나 Raspberry Pi 없이, Windows PC와 Arduino를 사용하여 Yahboom DOFBOT 로봇팔을 직접 제어하기 위한 솔루션입니다.DOFBOT 확장 보드(Expansion Board)는 기본적으로 I2C 통신을 사용하므로 PC와 직접 연결할 수 없습니다. 이 프로젝트에서는 Arduino를 USB-to-I2C 브릿지로 사용하여 PC의 Python 스크립트로 로봇팔을 정밀하게 제어합니다.🚀 주요 기능 (Key Features)PC 직접 제어: 고가의 임베디드 보드 없이 노트북/데스크탑에서 로봇팔 제어.GUI 컨트롤러: Tkinter 기반의 직관적인 슬라이더 제어 패널 제공.CLI 모드: 터미널 명령어를 통한 정밀 제어 지원.AI 연동 준비: OpenCV 및 YOLO와 연동하여 Visual Servoing 구현 가능.캘리브레이션 도구: 서보 모터의 영점 조절을 위한 미세 조정 툴 포함.🛠 하드웨어 준비물 (Hardware Requirements)Yahboom DOFBOT (Expansion Board + Robotic Arm)Arduino Uno (또는 Nano, Mega)PC (Windows 10/11 권장)12V 전원 어댑터 (DOFBOT 전원 공급용 필수)점퍼 와이어 (Male-to-Female) 3가닥USB 케이블 (Arduino 연결용)⚡ 배선 연결 (Wiring Diagram)가장 중요한 단계입니다. Arduino와 DOFBOT 확장 보드의 40핀 헤더를 아래와 같이 연결하세요.(DOFBOT 40핀 헤더의 핀 번호는 보드 구석의 작은 숫자를 참고하세요.)Arduino PinDOFBOT Pin기능 (Function)비고A4Pin 3SDA (Data)I2C 통신 데이터A5Pin 5SCL (Clock)I2C 통신 클럭GNDPin 6GND (Ground)필수 연결 (신호 기준점)⚠️ 주의: DOFBOT의 5V 핀(Pin 2, 4)과 아두이노의 5V를 연결하지 마세요. 전원 충돌로 보드가 손상될 수 있습니다.📦 소프트웨어 설치 (Installation)1. Arduino 설정Arduino_Bridge/Dofbot_Bridge.ino 파일을 엽니다. (파일명은 실제 업로드한 파일명에 맞게 수정 필요)Arduino IDE에서 보드와 포트를 선택합니다.코드를 업로드합니다.주의: 업로드 시 Wire.h 라이브러리가 필요합니다 (기본 내장).2. Python 환경 설정Python 3.8 이상이 필요합니다. 터미널에서 필요한 라이브러리를 설치하세요.Bashpip install pyserial opencv-python ultralytics tk
🎮 사용 방법 (Usage)1단계: 하드웨어 부팅배선이 올바른지 확인합니다.12V 어댑터를 DOFBOT에 연결하고 전원 스위치를 켭니다. (보드 LED 점등 확인)Arduino를 PC에 USB로 연결합니다.2단계: 컨트롤러 실행리포지토리를 클론하고 파이썬 스크립트를 실행합니다.(스크립트 내 PORT = 'COMx' 부분을 본인의 아두이노 포트 번호로 수정해야 합니다.)GUI 모드 (슬라이더 제어)Bashpython dofbot_gui.py
CLI 모드 (명령어 제어)Bashpython dofbot_cli.py
# 입력 예시: 1 90 (1번 모터 90도로 이동)

🔧 트러블슈팅 (Troubleshooting)Q. 코드를 실행했는데 모터가 움직이지 않아요.DOFBOT 보드 위의 **노란색 점퍼 캡(Servo Power Jumper)**이 꽂혀 있는지 확인하세요.12V 어댑터가 연결되어 있고 스위치가 켜져 있는지 확인하세요. (Arduino 전원만으로는 모터가 돌지 않습니다.)배선(SDA, SCL, GND)이 정확한지 다시 확인하세요.Q. Access Denied 또는 포트 에러가 떠요.Arduino IDE의 시리얼 모니터가 켜져 있다면 끄고 다시 실행하세요.Python 스크립트가 이미 실행 중인지 확인하세요.Q. 모터가 덜덜 떨리거나 이상한 곳으로 가요.dofbot_calib.py를 실행하여 물리적인 0점을 다시 맞춰주세요.
